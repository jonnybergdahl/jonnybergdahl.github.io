Detailed Description

MQTT transport adapter for PubSubClient.

PubSubClient is a synchronous, polling-based MQTT client.

Limitations:

  • The host firmware MUST call mqtt.loop() frequently.
  • QoS handling is best-effort.
  • Connection events are detected via rising-edge logic in tick().

◆ tick()

void PubSubClientTransport::tick ( )
inlineoverridevirtual

Detect MQTT connection transitions.

This method must be called periodically by the host firmware.

It detects a rising-edge transition (disconnected -> connected) and invokes the registered onConnect callback if present.

Note
This does NOT replace PubSubClient::loop().
